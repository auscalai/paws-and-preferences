## Introduction

Finding the perfect feline friend has never been easier. Paws and Preferences allows users to rapidly swipe left or right through an endless stack of cats served directly from the Cataas (Cat as a Service) API.

Designed with a **Mobile-First** approach and utilizing **Hammer.js** for native-feeling touch gestures, this application emphasizes performance through DOM pre-rendering and clean state management.

## Key Features

- **Fluid Card Stack UI:** Custom 3D-stacked interface with calculated scaling and depth effects.
- **Precise Swipe Gestures:** Integrated Hammer.js for responsive touch/mouse panning and velocity-based swipe recognition.
- **Keyboard Accessibility:** Full keyboard support (Arrow keys or A/D) with `tabindex` and `aria-*` compliance.
- **Dynamic Feedback:** Action buttons gradually scale and glow based strictly on drag progress (`deltaX`).
- **Optimized Rendering:** Elements are 100% pre-rendered in the DOM to eliminate layout shifts or flashing during rapid swiping.
- **Custom Design System:** Built utilizing glassmorphism, contextual shadows, and the `Fredoka` and `Nunito` font families.

## Installation Guide

Since the application uses no external build steps (Webpack, Vite, etc.), getting started is incredibly fast.

### Prerequisites

You simply need a web browser and a local file server.

### Setup Instructions

1. **Clone the repository:**
   ```bash
   git clone https://github.com/auscalai/paws-and-preferences.git
   cd paws-and-preferences
   ```

2. **Serve the project:**
   You can use any local web server. Here are two quick options:
   
   **Using Node.js (npx):**
   ```bash
   npx serve -l 3000
   ```
   
   **Using Python 3:**
   ```bash
   python -m http.server 3000
   ```

3. **Open the App:**
   Navigate to `http://localhost:3000` in your preferred web browser.

## Usage

Once the application loads:
- **Swipe Right** or click the **Mint Heart Button** (or press the `Right Arrow` / `D` key) to **Like** a cat.
- **Swipe Left** or click the **Coral Cross Button** (or press the `Left Arrow` / `A` key) to **Nope** a cat.
- Once you reach the end of the stack (15 cats), a summary gallery will render showcasing all of your chosen favorites.
- Click **Play Again** to fetch a fresh batch of cats.

## API Reference

We rely entirely on the **[Cataas (Cat as a Service) API](https://cataas.com/)** to populate the application data.

### Endpoints Used

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/cats` | Fetches a JSON array of cat metadata (IDs, tags, mimetypes). |
| `GET` | `/cat/{id}` | Direct URL mapping to display the parsed cat image. |

**Query Parameters Configuration:**
- `limit=15`: Request exactly 15 cats per batch to match our configured stack size.
- `skip={random}`: Generates a random integer offset to ensure the user gets fresh cats on every play session.

## Environment Variables

**No Environment Variables Required.**
  
Because Cataas is a public, open API, there are no API keys, client secrets, or `.env` files required to run this project. It is fully operable immediately after cloning.
